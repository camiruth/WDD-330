    Chapter 13: AJAX

  • Ajax is a technique that allows web pages to communicate asynchronously with a server, and it dynamically updates web pages without reloading.
  • A client, such as a web browser, will request a resource (usually a web page) from a server, which processes the request and sends back a response to the client.
  • The Fetch API provides a global fetch() method that only has one mandatory argument, which is the URL of the resource you wish to fetch.
  • each response object has an ok property that checks to see if the response is successful.
  • Redirect- used to redirect to another URL. Text- takes a stream of text from the response reads it to completion and then returns a promise that resolves to a USVSting object.
  • File responses- Blob method is used to read a file of raw data and returns as a blob object. json method transforms a stream of JSON data into a promise that resolves to a JS object.
  • Request Constructor Properties- URL, method, headers, mode, cache, credentials, redirect.
  • Fetch API is a living standard meaning it is being updated and changed all the time; experimental technology.
  • The Response interface allows you to control the response received from a request or to create your own response objects.
  • The Request interface allows you to create a request object that contains information about the request being made, such as the URL and headers.
  • The Headers interface allows you to create HTTP headers that can be added to a request or response object.
  • The FormData interface makes it easier to send data from forms.
