    Chapter 10 Notes: Testing and Debugging

  • System Error- there's a problem with the system or external devices with which the program is interacting.
  • Programming Error- thr program contains incorrect syntax or faulty logic; it could even be as simple as a typo.
  • User Error- the user has entered data incorrectly, which the program is unable to handle.
  • Stack trace is a sequence of functions or method calls that lead to the point where the error occurred.
  • Warnings will cause the program to crash as it will continue to run after the warning. Like assigning a value to a variable that is undeclared.
  • Sometimes Javascript fails silently in the background making errors hard to detect.
  • ECMAScript 5 catches errors that are more bad practice but won't actually break your code. Put.... 'use strict'; in the first line of the Javascript file.
  • JS Lint, JS Hint, and ES Lint can be used to catch other sloppy programming practices. Can be good to enforce a programming style guide but can be fairly unforgiving.
  • Feature Detection: best way to determine browser support for a feature. if(window.holoDeck){virtualRality.activate();}
  • Modernizr is a library that offers an easy way to implement feature detection.
  • Debugging is a process to find out where bugs occur, alert method to show certain points in the code as it stops the program from running until clicked okay. Console.log() is also a great way to debug.
  • Firefox, Edge, Chrome, Safari all have a debugging tool that will help you set break points in your code and see the values that those break points.
  • Also, you can use the keyword debugger; to create that breakpoint.
  • Try, catch, and finally are used as exception handling. Any code placed inside a try block will pass any error objects to a catch block when an exception occurs. Any code inside a finally block will run if an exception does or does not occur.
  • Test-driven development is the practice of writing tests that fail, then writing the code that passes the test, then refactoring the code every time a new feature is implemented.
  • The Jest framework can be used to test your code.
